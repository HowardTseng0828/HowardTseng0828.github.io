React + Django 環境架設

發表於 2023-05-12 分類於 React Frontend + Django Backend
文章字數： 1.9k 所需閱讀時間 ≈ 2 分鐘

前言

Frontend

前端框架的選擇對於項目的成功至關重要。在探索過程中發現， Vue 和 React 兩者都受到廣泛推薦。然而，最終決定採用 React 作為的前端框架，原因是 React 在開發社群中享有廣泛的支持和擁護，並且擁有龐大的生態系統。這些因素表明 React 是一個穩固、成熟且可靠的選擇。

雖然有部分人認為 React 的學習曲線較陡峭，但對技術的學習充滿熱情，並相信這將是一個有益的挑戰。學習 React 將提供更多靈活性和創造力，有助於將來開發更複雜的應用程序。而且，因為 Vue 和 React 在某些方面有相似之處，相信日後如果需要轉向 Vue ，這些共通點將使過渡變得更加容易。

Backend

在選擇後端框架時，希望找到一個功能豐富且強大的解決方案，以滿足比價網站的複雜需求。經過對不同框架的評估，最終選擇了 Django 作為後端框架。

Django 是一個成熟且廣受歡迎的 Python Web 框架，它提供了強大的功能，如內建的數據庫 ORM 、管理後台、用戶身份驗證等。 Django 的優雅設計和易於擴展的特性將帶來更高效且流暢的開發體驗。

Django 的豐富功能契合了比價網站的需求，相信它將成為構建強大後端的理想選擇。同時， Django 在開發社群中擁有龐大的支持和資源，這將使開發過程中能夠快速解決問題並取得協助。

SlashCommand (/)指令

SlashCommand 優點

  • 直觀易用：簡單且直觀的方式，可以輕鬆地定義和處理斜線指令
  • 增強互動性：使用者只需在聊天中輸入斜線指令，即可觸發特定功能或操作
  • 減少誤操作：由於斜線指令具有明確的語法，有助於減少誤操作和提供更一致的使用體驗
  • 自動完成和參數驗證：Discord 平台提供自動完成和參數驗證的功能，確保輸入的正確性
  • 支援互動元件：可以與 Discord 的互動元件（例如按鈕、下拉選單等）結合使用

總結而言，SlashCommand 提供了直觀易用、增強互動性、減少誤操作、自動完成和參數驗證，以及支援互動元件等優點，使得開發者能夠更輕鬆地實現功能豐富且交互性強的機器人應用
